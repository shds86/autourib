== 1. Системные требования ==



  -> Минимальные требования:

      - Любая операционная система семейства Windows NT

      - Java Runtime Environment 6u18



== 2. Установка  ==



Для программы пока не планируется создание инсталлятора. На данный момент программа запускается щелчком по файлу AvtoURIB.jar, также возможен запуск с ярлыка или через автозапуск при старте системы.



== 3. Использование ==



При первом запуске программа рядом с собой создает папку имени себя, в которой создается пустой файл настроек AutoURIB.ini, если его нет. Или загружает настройки из уже существующего файла настроек.

Первое, что необходимо сделать, это перейти на вкладку Настройки и заполнить ВСЕ поля. Если FTP не используется, то заполнить относящиеся к нему поля можно любой билибердой, главное, чтобы было.

Далее нажатием кнопки Применить проиходит сохранение настроек в файл настроек. Программа проверяет наличие/отсутствие/правильность заполнения файла настроек.

Программа может быть свернута в трей пользователем по кнопке [Х], может также запускаться свернутой в трей, если в настройках это указано.

Есть возможность очистить лог, если щелкнуть по нему RMB.



== 4.1 Информация о текущей версии ==

В версии 1.5 исправлена ошибка, связанная с расписанием. Теперь можно создавать несколько расписаний. Появилось 2 типа расписаний: обмен и синхронизация папок. Это очень удобно при синхронизации папок ExtForms и PrnForms, если после обновления какого-либо отчёт или обработки требуется обновить его в большом количестве удалённых филиалов.

Галка «Эксперт-режим» переключает оформление вкладки «Главная». При установленной опции «Эксперт-режим» пункты «Загрузка» и «Выгрузка» разбиваются на более детальные подпункты.



== 4.2 Информация о предыдущих версиях ==

В версии 1.1 добавлена возможность создавать, сохранять расписание, запускать обмен по расписаниям.

В данной версии не работает синхронизация содержимого папок, не смотря на то, что создана отдельная вкладка.
